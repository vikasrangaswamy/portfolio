Backtracking is the choose → explore → unchoose pattern. Always undo (pop) to restore state for the next branch. Pass a "start index" to avoid duplicates in combinations, and prune early when a partial solution violates a constraint. Copy your path when saving the answer because you mutate it during recursion.

## Representative approach

```python
def subsets(nums):
    """Generate all subsets of nums (power set)."""
    result = []
    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()
    backtrack(0, [])
    return result

def combinations(n, k):
    """Generate all combinations of k elements from 1..n."""
    result = []
    def backtrack(start, path):
        if len(path) == k:
            result.append(path[:])
            return
        for i in range(start, n + 1):
            path.append(i)
            backtrack(i + 1, path)
            path.pop()
    backtrack(1, [])
    return result
```

> **Tip:** Backtracking template — choose → recurse → unchoose. Prune early when a constraint is already violated.
