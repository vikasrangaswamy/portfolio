Always use a dummy node to simplify head updates and handle edge cases. Fast/slow pointers are your friend for cycle detection and finding the middle. Remember to set `.next = None` when breaking cycles.

## Representative approach

```python
def reverse(head):
    """Reverse a singly linked list iteratively."""
    prev = None
    curr = head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev

def detect_cycle(head):
    """Detect cycle using fast and slow pointers."""
    slow, fast = head, head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False
```
