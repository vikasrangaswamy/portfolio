Tree problems split naturally into traversal, path-sum, height, view, and BST-specific patterns. Prefer recursion when you need "info from subtrees," and use explicit stack/queue for control. Draw small examples: empty, single node, left-only, right-only.

## Traversal patterns

| Traversal | Order | Use case |
|-----------|-------|----------|
| Inorder | Left → Root → Right | BST sorted output |
| Preorder | Root → Left → Right | Tree serialization |
| Postorder | Left → Right → Root | Delete tree, evaluate expression |
| Level order (BFS) | Level by level | Min depth, connect nodes |

## Representative approach

```python
def inorder_traversal(node):
    """Inorder DFS: Left → Root → Right (BST gives sorted)."""
    if not node:
        return []
    result = []
    def traverse(n):
        if not n:
            return
        traverse(n.left)
        result.append(n.val)
        traverse(n.right)
    traverse(node)
    return result

def level_order_traversal(root):
    """BFS level order; useful for min depth, row max, etc."""
    if not root:
        return []
    queue = [root]
    result = []
    while queue:
        node = queue.pop(0)
        result.append(node.val)
        if node.left:
            queue.append(node.left)
        if node.right:
            queue.append(node.right)
    return result
```
